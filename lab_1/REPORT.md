### 2.2. Полиморфизм через VTable

Полиморфное поведение реализовано через таблицу виртуальных функций (VTable) — структуру `AlgebraOperations`:

```c

typedef struct AlgebraOperations {
// Операции  кольца
	void (*addFn)(const void*, const void*, void*);
	void (*subtractFn)(const void*, const void*, void*);
	void (*negateFn)(const void*, void*);
	void (*zeroFn)(void*);
	void (*multiplyFn)(const void*, const void*, void*);
	void (*oneFn)(void*);
	// Проверки
	int (*isZeroFn)(const void*);
	int (*isOneFn)(const void*);
} AlgebraOperations;

```

Для  каждого  типа  данных (`Integer`, `Double`, `Complex`) создаётся  свой  экземпляр VTable через  функции `GetIntegerOps()`, `GetDoubleOps()`, `GetComplexOps()`.

### 2.3. Структура матрицы

```c
typedef struct Matrix {
	void* data;  // Указатель на данные (полиморфный)
	int size;  // Размер n×n
	AlgebraOperations* operations;  // Указатель  на VTable типа
	size_t element_size;  // Размер элемента в байтах
} Matrix;

```

## 5. Проверка аксиом кольца

Для каждого типа данных реализована проверка 7 аксиом алгебраического кольца:

1.  **Замкнутость сложения** — результат сложения принадлежит тому же типу

2.  **Ассоциативность сложения** — `(a + b) + c = a + (b + c)`

3.  **Коммутативность сложения** — `a + b = b + a`

4.  **Нейтральный элемент сложения** — существует `0`: `a + 0 = a`

5.  **Аддитивный обратный** — существует `-a`: `a + (-a) = 0`

6.  **Нейтральный элемент умножения** — существует `1`: `a × 1 = a`

7.  **Дистрибутивность** — `a × (b + c) = a × b + a × c`

**Результат:** Все три типа (`Integer`, `Double`, `Complex`) успешно проходят проверку всех 7 аксиом.

## 6. Дополнительные задания

### 6.1. LU-разложение

**Алгоритм:** Метод Краута (Doolittle) с единичной диагональю в L.

**Реализация:**

- Функция `matrix_lu_decompose(A, L, U)` разлагает матрицу A на нижнюю (L) и верхнюю (U) треугольные матрицы

- Проверка на вырожденность (сингулярность) по диагональным элементам U

- Поддержка Integer→Double конвертации (так как L содержит дробные коэффициенты)

**Код ошибки:** `ERR_SINGULAR_MATRIX` при обнаружении нулевого элемента на диагонали.

---

### 6.2. QR-разложение

**Алгоритм:** Modified Gram-Schmidt.

**Реализация:**

- Функция `matrix_qr_decompose(A, Q, R)` разлагает матрицу A на ортогональную (Q) и верхнюю треугольную \(R\)

- Ортогонализация столбцов с нормализацией

- Проверка на линейную зависимость столбцов

**Свойства:**

- `Q^T × Q = I` (ортогональность)

- `A ≈ Q × R` (восстановление исходной матрицы)

---

### 6.3. Решение СЛАУ

Реализовано два метода решения системы `A × x = b`:

#### Метод 1: Через LU-разложение

```c

ErrorCode solve_lu(const Matrix* A, const Matrix* b, Matrix* x);

```

**Алгоритм:**

1.  LU-разложение: `A = L × U`

2.  Прямой ход: `L × y = b` (нахождение промежуточного вектора y)

3.  Обратный ход: `U × x = y` (нахождение решения x)

#### Метод 2: Через QR-разложение

```c

ErrorCode solve_qr(const Matrix* A, const Matrix* b, Matrix* x);

```

**Алгоритм:**

1.  QR-разложение: `A = Q × R`

2.  Умножение на Q^T: `R × x = Q^T × b`

3.  Обратная подстановка для верхней треугольной системы

---

### 6.4. Бенчмарк: LU vs QR

Реализована функция сравнения производительности `benchmark_lu_vs_qr(int size)`:

| Размер матрицы | LU (мс) | QR (мс) | Отношение |

| 3×3 | ~0.010 | ~0.030 | ~3.0x |
| 5×5 | ~0.025 | ~0.075 | ~3.0x |

**Вывод:** QR-разложение примерно в 3 раза медленнее LU, но более численно устойчиво для плохо обусловленных матриц.

## 7. Система обработки ошибок

Вместо `fprintf(stderr)` реализована система кодов ошибок `ErrorCode`:

```c

typedef enum {
	ERR_OK = 0,
	ERR_NULL_POINTER = -1,
	ERR_SIZE_MISMATCH = -2,
	ERR_SINGULAR_MATRIX = -3,
	ERR_UNSUPPORTED_OPERATION = -4,
	ERR_TYPE_MISMATCH = -5,
	ERR_OUT_OF_MEMORY = -6,
	ERR_INVALID_INPUT = -7
} ErrorCode;

```

**Принцип  работы:**

- Библиотечные функции (`matrix.c`, `algebra.c`) возвращают коды ошибок

- Интерфейс (`interface.c`) преобразует коды в читаемые сообщения через `error_message()`

- Тесты проверяют коды ошибок через `TEST_ASSERT(err == ERR_OK, ...)`
